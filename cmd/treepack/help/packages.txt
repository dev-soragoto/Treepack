Packages

Packages describe source assets and where they are installed in the staged
output. Supported source prefixes are file:, url:, and github:.

An asset pattern selects a source asset. It does not select entries inside a zip.
Zip entries are installed by one asset with install = "extract" followed by one
or more [[packages.steps]] operations that copy from extract/<asset-name>/.
Portable zip-name corrections are also applied to literal step paths below
extract/<asset-name>/. cp_regex instead matches the corrected physical names.
Unsafe paths, special files, and portable-name conflicts reject the whole zip.

url: accepts exactly one source asset, declared with asset or one [[packages.assets]]
entry. file: pointing directly at a regular file may omit asset; when asset is
present, its regex must MatchString the file basename. Regexes are not implicitly
anchored. file: directories and GitHub releases require every pattern to resolve
to exactly one source asset.

install = "extract" cannot be combined with target; extracted entries are moved
to package output by steps. Packages default to required = true. A step without
required inherits its package value, while an explicit step value overrides it.
An optional step failure is recorded as a warning and later steps continue; the
package's existing output is still merged. A required step failure stops that
package, but only a required package blocks the complete build.
